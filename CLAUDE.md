# Rust/Wasm Streaming UI Demo

A learning demo mirroring Prime Video's architecture: the **entire UI** is written in
Rust, compiled to WebAssembly, and drawn to a single WebGL2 canvas. **No DOM for UI.**
JS is a thin glue layer only (no React). Full spec in `PRD-rust-wasm-streaming-demo.md`.

## Architecture
Rust/Wasm owns: app state machine (Home/Meta/Player), layout, focus model, animation,
the sprite batcher, all WebGL2 draw calls, and the player UI overlay.
JS glue owns only host-reachable things: the requestAnimationFrame loop, keydown
forwarding, image decode (`createImageBitmap`) → texture pixels, `fetch('content.json')`,
and `<video>` control. Video plays in a native `<video>` behind a transparent canvas.

## Stack
- Rust (stable) + wasm-pack / wasm-bindgen
- web-sys / js-sys for host bindings
- `glow` crate for WebGL2 + a hand-written sprite batcher
- serde / serde_json for content
- Served with **python3 -m http.server** (NOT Vite — local Node is v11, too old)

## Files
- `src/lib.rs` — the Rust engine (the heart of the project)
- `Cargo.toml` — crate config + pinned deps
- `index.html` — single `<canvas id="app">`
- `main.js` — thin glue: init wasm, size canvas, call `start()`
- `pkg/` — GENERATED by wasm-pack; never edit by hand

## Environment notes

- Deps pinned for Rust 1.76: wasm-bindgen 0.2.92, web-sys 0.3, glow 0.13.

## Scale (paginated catalog + 2D texture window)
- Content is paginated: content.json = featured + first 10 rails + `pageCount`; rails-1..4.json
  are fetched ON DEMAND. 50 rails x 20 cards total.
- Rust: `append_rails(json)` grows content.rails (+ extends card_focus/anim_x);
  `loaded_rail_count()`, `focus_rail()` getters drive JS prefetch.
- JS (main.js): `maybeFetchNextPage()` prefetches the next page when
  focus_rail >= loaded_rail_count - 3; `renderRailTitles()` renders only new titles.
- Textures use a 2D window: `window_posters()` keeps rails within LOAD_RADIUS(2) AND only
  cards within HSPAN(6) of each rail's focused card. `free_offscreen()` frees the rest.
- Drawing already virtualizes (culls off-screen rails+cards) so 50 rails == ~same per-frame work.

## Convention
This is a teaching codebase: keep `lib.rs` readable and well-commented, prefer clarity
over cleverness, and keep `main.js` minimal. Build & verify each milestone in the
browser before starting the next.
